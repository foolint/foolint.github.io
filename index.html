路由

基本信息

一个固定不变的路由器, 配合各种可自定义的路由协议, 来实现灵活多变的路由策略.
即容器固定，装什么就看自己的心情了。

路由组件有两个部分：路由器(Yaf_Router)和路由协议(Yaf_Route_Abstract).

最后注册的路由协议, 最先尝试路由

默认情况下,我们的路由器是Yaf_Router, 而默认使用的路由协议是Yaf_Route_Static,是基于HTTP路由的, 它期望一个请求是HTTP请求并且请求对象是使用Yaf_Request_Http

路由协议分类：what

1、Yaf_Route_Simple 会匹配请求中的query string，然后找到路由信息; 你需要做的只是告诉 Yaf_Route_Simple，在$_GET中哪个是Module，哪个是Controller，哪个是Action

2、Yaf_Route_Supervar ?r=/controller/action/params/value 这种的形式。

3、Yaf_Route_Static ，
Yaf_Router 只有一个Yaf_Route_Static 作为它默认的路由.实例化 Yaf_Route_Static 是没有必要的，也没必要将它加入Yaf_Router的路由堆栈，因为在Yaf_Router的路由堆栈中总是存在它的一个实例，并且总是在最后被调用。default 值。
形式是：/yafmodule/yafcontroller/yafaction?tkey1=tval1&tkey2=tval2 这种正常的 queryString 形式。

4、Yaf_Route_Map 提供特殊字符分割取 controller action.

5、Yaf_Route_Rewrite 更严格的自定义。

6、Yaf_Route_Regex Yaf内置的路由中最灵活的，正则过滤。

7、自定义路由：Yaf_Route_Interface是Yaf路由协议的标准接口,

它的存在使得用户可以自定义路由协议。

路由协议添加: how

1、实例化、执行调用，手动调用。

2、在配置文件中初始化(application.ini)，自动调用。

切片是对变量的引用吗
并不是真正意义上的动态数组，而是一个引用类型。slice总是指向一个底层array，slice的声明也可以像array一样，只是不需要长度。
是不是进行了强制类型转化成了数组？ s[1:] 的类型是 string.

关于赋值
两种形式就够用了，其他看懂就行。
var vname1, vname2, vname3 type= v1, v2, v3 //全局变量。
vname1, vname2, vname3 := v1, v2, v3 //函数内部变量，编译器推倒相应类型。
全局变量与函数内部变量的区别：全局变量可以在函数外编译通过。

由于长度也是数组类型的一部分，因此[3]int与[4]int是不同的类型，数组也就不能改变长度。
数组之间的赋值是值的赋值，即当把一个数组作为参数传入函数的时候，传入的其实是该数组的副本，而不是它的指针。
如果要使用指针，那么就需要用到后面介绍的slice类型了

a := [3]int{1, 2, 3} // 声明了一个长度为3的int数组
b := [10]int{1, 2, 3} // 声明了一个长度为10的int数组，其中前三个元素初始化为1、2、3，其它默认为0
c := […]int{4, 5, 6} // 可以省略长度而采用...的方式，Go会自动根据元素个数来计算长度

slice, 动态数组（不初始化长度），对数组的引用（指针）
